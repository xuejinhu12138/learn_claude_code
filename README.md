# 关于我如何生成这个claude.md任务清单

我是一个新手，学习过前后端但是没学习过这里面的语言，也不了解整个claude-code的设计架构细节，你可以理解为我是一个有编程经验但是在这个领域完全空白的小白。我要学习这里面claude-code的代码设计、实现。我希望我最终的目标是独立写出一个这样完整的项目，预计时间是1周半。现在项目里有src是源码，claude.md是当时用于CFP的，请你帮我改造成我需要的md（中文模式），其中这里面很多是提问的方式，我希望的是我需要提问加写代码的方式，所有的语法，设计等都是通过提问+预定义目标+我本人手动写代码实现编译test测试通过+你评价修正建议的形式。

在claude.md里你没有用上sessions和progress文件对我的学习情况进行跟踪记录，并且学习计划过于宽泛（如`main.tsx` 启动时做了哪些事情？（从第 1 行到 REPL 启动，列出关键步骤） 我根本就不了解所有的语法，也不可能一下子看得懂所有的代码，一个文件夹3000行不是一个小白能一下子看懂的），任务跟踪也没做到，而且第一天里也是这样，我都不会语法怎么完成任务，你这里的任务定义应该是抽象（不带有代码细节）但是又具体，如需要掌握XX函数。。。任务不是按天分配而是按照掌握能力，并且 我到时候的方式是 我们现在学到哪里了，请告诉我下一步是什么，你的回答模式应该是先介绍语法/设计架构思想，然后给我分配任务让我实现代码！

现在的改造非常好，但是学习路线图还少了一些。我举个例子hook等等，也就是说你没有看src完全的代码，在src下有那么多的文件夹，如果最后要让我能实现，这里面每一个文件夹内涉及的内容你都需要教给我！所以也就是说，随着我们学习的逐渐完善，我们自己实现的工程里也会逐步扩展成他src那个样子。所以请先自己完整的阅读src的代码，梳理清楚代码流程，确保教给我的学习路线图是正确的，逐步递增的！ 所以你看完后主需要改造学习路线图就行，因为涉及代码量较大，你可以创建一个梳理的文件夹，梳理清楚每个文件夹具体哪些可以教怎么教拆分一下，最后整理好顺序合并到学习路线图里。
