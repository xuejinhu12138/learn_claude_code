#### P1-8 全局进程状态（Bootstrap 单例）

- 学到顺序规则：Schema → type → 变量 → 函数（从上到下依赖顺序）

- 写了 `history.test.ts`，`bun test` 全绿（6 pass）

#### P2-6 真正的 API 调用
- 发现本地服务是 OpenAI 兼容格式（非 Anthropic），安装 `openai` SDK
- 本地模型：omlx 服务，`Qwen3.5-0.8B`，key: `xuejinhu`，地址: `http://127.0.0.1:8000/v1`
- 创建 `api.ts`，实现 `sendMessage(messages: Message[]): Promise<string>`

- 类型兼容：`messages as ChatCompletionMessageParam[]` 强转

## 遇到的困难

1. **`text` vs `content` 字段**：TextBlock 的字段是 `text` 不是 `content`，反复出现 3 次才记住
2. **Zod 的必要性**：一开始不理解为什么不直接用 `type`，通过实际崩溃演示后理解
3. **模块加载顺序**：`import` 触发模块级代码执行，此时 `initState` 还没跑 → 解法：懒初始化
4. **DEFAULT_BASE_URL 重复声明**：在 constants 里不小心写了两次同名常量
