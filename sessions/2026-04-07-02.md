## 本次学习内容

### Phase 3 多轮对话实现（P3-1 ~ P3-3 补全）

#### P3-1 readline 交互式输入
- 用 `readline.createInterface` 创建终端输入接口
- **关键概念**：`rl.question` 是异步回调，不能在 `while(true)` 里直接用
- **解法**：封装 `prompt()` 函数返回 Promise，用 `await` 真正暂停循环
- 踩坑：`while(true) + rl.question` 会瞬间注册无数回调导致死循环
- 踩坑：`readline` 在模块级创建导致非 chat 命令也不退出 → 移到 `case 'chat'` 内部

#### P3-2 System Prompt 注入
- 在 `constants/index.ts` 定义 `SYSTEM_PROMPT`，包含当前时间和工作目录
- `sendMessage` 里在消息数组最前插入 system 消息
- system 消息不存入 `getHistory()`，每次请求动态注入（设计正确）
- 验证：问"今天是几号"，模型正确回答 2026年4月7日

#### P3-3 优雅退出
- `process.on('SIGINT', ...)` 捕获 Ctrl+C
- 错误时 `catch` 后继续循环，不退出
- 清理调试用的 `console.log`

### Phase 3 补全（P3-4, P3-7, P3-8）

#### P3-4 JSON Schema 基础
- 理解 JSON Schema 是"描述数据形状的元数据"
- 在 `types/tool.ts` 定义 `JSONSchemaProperty`、`JSONSchema`、`Tool` 类型
- 理解为什么工具定义用 `type` 不用 Zod：自己写的代码编译期 TS 保证，外部数据才需要 Zod
- 写了 `read_file` 工具示例验证类型正确

#### P3-6 流式输出（Streaming）
- `stream: true` 开启流式
- `for await...of` 逐块处理
- `process.stdout.write(token)` 不换行追加（打字机效果）
- 流结束后 `write('\n')` 换行
- `text += token` 收集完整回复存入历史
- 踩坑：`process.stdout.write("AI> \n")` 有换行导致回复另起一行，去掉 `\n`

#### P3-7 stop_reason 处理
- `finish_reason` 在流式最后一个 chunk 的 `choices[0].finish_reason`
- 定义 `SendMessageResult = { text: string; stop_reason: string }`
- `sendMessage` 返回包含 `stop_reason` 的结果对象
- `printDebug` 打印 stop_reason，验证输出 `"stop"` 正常
- 拼写踩坑：`stop_reasion` → `stop_reason`（多写了个 i）

#### P3-8 错误处理与重试
- 在 `utils/retry.ts` 实现 `withRetry<T>(fn, maxRetries, delayMs)`
- 指数退避：每次失败后 delay 翻倍
- `await new Promise(r => setTimeout(r, delayMs))` 暂停 N 毫秒的标准写法
- 测试：用 `callCount` 变量模拟前两次失败第三次成功
- 测试 Promise reject：用 `.rejects.toThrow()` 而不是 `toThrowError()`
- 踩坑：`1/0` 在 JS 不报错返回 `Infinity`，不能用来模拟失败

## 遇到的困难

1. **while(true) + rl.question 死循环**：回调异步，while 瞬间注册无数个监听
2. **readline 在模块级创建**：导致所有命令都不退出，移到 case 内解决
3. **stop_reasion 拼写错误**：typo 导致类型不匹配
4. **`1/0` 不报错**：JS 里除以0返回 Infinity，不是异常
5. **`.rejects.toThrow()` vs `toThrowError()`**：测试 Promise reject 要用前者
