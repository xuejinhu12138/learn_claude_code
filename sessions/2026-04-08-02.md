### P6-2：全局状态（Store + AppState）

  - `AppState` 类型：`messages / isLoading / inputValue / streamingText`
  - `appStore` 单例导出

### P6-3：useStore / useAppState Hook
- 新建 `src/ui/useAppState.ts`
- `useStore<T>(store)`: `useState(0)` + `useEffect` 订阅 + cleanup 取消订阅
- `useAppState()`: 无参数版本，绑定 `appStore`
- 学到：`const [, rerender] = useState(0)` 忽略值只取更新函数；`rerender(n => n+1)` 触发重渲染的惯用法

### P6-4：消息列表组件
- `<MessageList>` 组件：`useAppState()` 读 `messages`，`map` 渲染每条消息
- `isLoading` 时末尾追加 `{role:'assistant', text: streamingText}` 占位
- 演示了 `setInterval` + `useRef` 计数驱动消息逐条出现的动画效果

### P6-5：输入组件
- `<StatusBar>` 组件：`useInput` 处理键盘，`useAppState()` 读 `inputValue`
- 关键 bug 修复：JSX 里用 `appStore.get()` 不触发重渲染，必须用 `useAppState()` 才能响应变化
- 处理：普通字符追加 / Backspace 删除 / Enter 发送 / 空输入框时 q/Escape 退出

### P6-6：流式输出接入真实 AI
- Enter 后调用 `runAgent(currentInput, depsAgent)`（async IIFE 模式）
- `depsAgent` 提到模块顶层（只创建一次），避免每次重渲染重建
- `api.ts` 修改：`process.stdout.write(token)` → `appStore.set(prev => ({...prev, streamingText: text}))`，实现流式文字更新到 UI
- `agent.ts` 移除 `process.stdout.write("AI> ")`，避免和 Ink 渲染冲突
- `runAgent` 完成后读 `depsAgent.getHistory()` 最后一条 assistant 消息，追加到 `messages`

## 关键调试记录

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `Invalid hook call` | `useInput` 在组件外（模块顶层）调用 | 移进函数组件体内 |
| 输入字符界面不更新 | JSX 里用 `appStore.get()` 直接读，不触发重渲染 | 改为 `useAppState()` 订阅 |
| 终端输出双重显示 | `createDefaultDeps()` 写在 StatusBar 函数体内，每次重渲染都新建 deps | 移到模块顶层，只创建一次 |
| Ink 渲染和流式输出混乱 | `api.ts` 的 `process.stdout.write` 直接写终端，和 Ink 冲突 | 改为更新 `appStore.streamingText` |
| `export type Store` 报错 | 类是值+类型，不能用 `export type` | 改为普通 `export class` / `export { Store }` |

## 架构设计备注

**当前设计取舍**：`api.ts` 直接依赖 `appStore`（UI 层），存在层级倒置。
- 现状：底层 API 模块 → 依赖 → 上层 UI Store
- 理想：API 模块通过回调/事件通知，不直接引用 UI 模块
- 后续可重构：给 `sendMessage` 加 `onToken?: (token: string) => void` 回调参数
