## 架构亮点

### ✨ 优雅的"等待用户输入"实现

```typescript
// 问题：JavaScript 没有"同步阻塞等待输入"
// 方案：用 async/await + Promise 模拟同步行为

const confirmed = await new Promise<boolean>((resolve) => {
    // 将 resolve 函数"藏"在回调里
    appStore.set({
        pendingConfirmation: {
            onConfirm: () => resolve(true),   // 用户按 Y → 解锁，返回 true
            onCancel: () => resolve(false)    // 用户按 N → 解锁，返回 false
        }
    });
});

// 👆 这行代码会"暂停"，直到用户按键触发 resolve()
// 👇 confirmed 拿到 true 或 false，决定是否继续

if (!confirmed) return '取消';
execSync(command);  // 只写一次！
```

这是 **异步编程中模拟同步等待的经典模式**！

## 知识点总结

### 1. React Props 系统
- Props 是父组件传递给子组件的参数对象
- 类型定义：`type Props = { field: type }`
- 解构取值：`function Component({ field }: Props)`
- 函数类型：`() => void`（无参无返回）、`(x: T) => void`（有参无返回）

### 2. async/await + Promise 配合
```typescript
// 创建可手动控制的 Promise
const result = await new Promise<T>((resolve, reject) => {
    // 异步操作完成时调用 resolve(value)
    // 失败时调用 reject(error)
});
```

### 3. 提前返回（Early Return）避免嵌套
```typescript
// ❌ 嵌套难读
if (isDangerous) {
    if (confirmed) {
        execSync(command);
    }
} else {
    execSync(command);
}

// ✅ 提前返回
if (isDangerous && !confirmed) return '取消';
execSync(command);  // 只写一次
```

### 4. JSX 条件渲染
```typescript
{condition && <Component />}       // true 时渲染
{condition ? <A /> : <B />}        // 三元表达式
{items.map(item => <Item />)}      // 列表渲染
```
