## 核心知识点

### 1. React 定时器的生命周期

```typescript
useEffect(() => {
    const timer = setInterval(() => {
        setFrame(prev => (prev + 1) % frames.length);
    }, 100);

    // ⚠️ 组件卸载时必须清理，否则内存泄漏
    return () => clearInterval(timer);
}, []);  // 空依赖 = 只在挂载时执行一次
```

**为什么必须清理？**
- 组件卸载后，定时器还在运行
- 定时器回调访问已销毁的组件状态 → 内存泄漏
- 多次挂载/卸载 → 多个定时器叠加 → 卡顿

### 2. 模运算实现循环

```typescript
// i = 0, 1, 2, ..., 9, 10, 11, ...
// i % 10 = 0, 1, 2, ..., 9, 0, 1, ...
const index = (prev + 1) % array.length;
```

### 3. 测试隔离 — 单例状态的陷阱

```typescript
// ❌ 错误：多个测试共享 appStore，状态污染
test('test1', () => {
    appStore.set({ value: 1 });
    expect(appStore.get().value).toBe(1);
});

test('test2', () => {
    // ⚠️ 这里 value 可能还是 1！
    expect(appStore.get().value).toBeUndefined();  // 失败！
});

// ✅ 正确：每个测试前重置状态
beforeEach(() => {
    appStore.set(() => initialState);
});
```

### 4. 同时满足多个需求

```typescript
// 旧代码：只更新 UI，测试失败
if (tokenCount > threshold) {
    appStore.set({ showWarning: true });
}

// ✅ 新代码：既调用回调（测试需要），又更新 UI（界面需要）
if (tokenCount > threshold && !warningIssued) {
    this.onWarn("警告");  // 给测试验证用
    appStore.set({ showWarning: true });  // 给 UI 显示用
    this.warningIssued = true;
}
```

## 架构亮点

### ✨ 状态驱动的 UI 更新

```
Agent 执行状态变化
      ↓
appStore.set({ currentStatus: "正在思考..." })
      ↓
所有订阅了 useAppState() 的组件重渲染
      ↓
Spinner 组件检测到 currentStatus 不为空
      ↓
条件渲染：显示旋转动画 + 状态文字
```

这是 **单向数据流**的经典应用！

### ✨ 定时器 + 状态驱动动画

```typescript
// 定时器负责"推进时间"
setInterval(() => {
    setFrame(prev => prev + 1);  // 每 100ms frame++
}, 100);

// 状态变化触发重渲染，显示不同字符
<Text>{frames[frame % frames.length]}</Text>
```

分离关注点：
- **定时器**：控制时间流逝
- **状态**：存储当前帧索引
- **渲染**：根据索引显示字符
