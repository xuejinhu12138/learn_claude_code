#### 4. 深入学习 React Hooks 核心概念

**useState vs useRef**：
- `useState`：改变值触发重渲染，适合 UI 数据
- `useRef`：改变值不触发重渲染，适合内部状态（定时器 ID、最新回调等）

**闭包陷阱**：
- 定时器回调函数"记住"创建时的变量值
- 即使变量后来变化，回调访问的还是旧值
- 解决方案：用 `useRef` 存储最新值，回调中访问 `ref.current`

**为什么分两个 useEffect**：
- 第一个：每次 callback 变化时更新 `ref.current`，不重启定时器
- 第二个：只在 delay 变化时重启定时器，callback 变化不影响
- 分工明确，性能更好

**Hook 的独立性**：
- 每次调用 Hook 都创建独立的实例
- 不同组件调用同一个 Hook，互不干扰
- React 用调用顺序区分 Hook 实例

#### 5. 架构重构：解决 api.ts 的层级倒置

**问题**：`api.ts`（底层）直接依赖 `appStore`（UI 层）

**解决方案**：依赖注入（回调函数）
- `sendMessage` 新增可选参数 `onToken?: (token, fullText) => void`
- API 层通过回调通知外部，不直接操作 UI
- `agent.ts` 中传入回调：`sendMessage(messages, (token, text) => appStore.set(...))`

**重构后的架构**：
```
UI 层 (agent.ts)
  ↓ 传入回调函数
API 层 (api.ts)
  ↓ 调用回调通知
UI 层 (appStore 更新)
```

依赖方向正确：UI → API，而不是 API → UI

## 核心知识点

### 1. React Hooks 的本质

**Hook = 可复用的逻辑模板 + 自动生命周期 + 响应式状态**

| 特性 | Hook | 传统类 |
|------|------|--------|
| 代码复用 | ✅ | ✅ |
| 自动清理 | ✅ | ❌ |
| 自动重渲染 | ✅ | ❌ |
| 可组合 | ✅ 强 | ⚠️ 弱 |

### 2. 闭包陷阱的本质

```typescript
// 问题：定时器回调"记住"创建时的值
useEffect(() => {
    setInterval(() => {
        console.log(count);  // 永远打印初始值
    }, 1000);
}, []);  // 空依赖 = 只执行一次

// 解决：用 ref 存储最新值
const countRef = useRef(count);
useEffect(() => { countRef.current = count; }, [count]);
useEffect(() => {
    setInterval(() => {
        console.log(countRef.current);  // 总是最新值
    }, 1000);
}, []);
```

### 3. useEffect 依赖数组的规则

```typescript
useEffect(() => {
    // 副作用代码
}, [dep1, dep2]);  // dep1/dep2 变化时重新执行

// 常见模式：
[], // 只在挂载时执行一次
[count], // count 变化时执行
// 不写依赖 = 每次渲染都执行（通常不需要）
```

### 4. Hook 的独立性

```typescript
function ComponentA() {
    useInterval(() => {}, 1000);  // 创建实例 A
}

function ComponentB() {
    useInterval(() => {}, 2000);  // 创建实例 B（独立）
}

// 两个实例互不干扰，各自管理定时器
```

### 5. 依赖注入 vs Hook

| 场景 | 方案 | 原因 |
|------|------|------|
| 组件间复用 | Hook | 可以调用其他 Hook |
| 模块间解耦 | 依赖注入 | 不能在模块中调用 Hook |
| 底层不依赖上层 | 依赖注入（回调） | 保持依赖方向正确 |

### ✨ 依赖注入的价值

**重构前**：
```
API 层 ←依赖← UI 层  ❌ 反向依赖
```

**重构后**：
```
API 层 ←调用← UI 层  ✅ 正确方向
  ↓
回调通知
  ↓
UI 层更新
```

**优点**：
1. API 层可独立测试
2. API 层可用于其他场景（CLI / Web / Desktop）
3. 符合"依赖倒置原则"
