## 核心知识点

### 1. 命令模式（Command Pattern）

```typescript
type Command = {
    name: string;
    description: string;
    execute: (args: string, deps?: AgentDeps) => Promise<string>;
};

// 注册
commandRegistry.register(helpCommand);

// 执行
const result = await command.execute(args, deps);
```

**优点**：
- 解耦：命令定义和执行分离
- 可扩展：轻松添加新命令
- 可测试：每个命令独立测试

---

### 2. 双重状态同步问题

**问题**：两套数据不同步

```
UI 数据（简化）    Agent 数据（完整）
appStore.messages  depsAgent.getHistory()
     ↓                    ↓
  显示在界面          发给 LLM
```

**解决**：操作时同时更新

```typescript
// ❌ 错误：只更新一个
appStore.set({ messages: [] });

// ✅ 正确：同时更新
appStore.set({ messages: [] });
deps.clearHistory();
```

---

### 3. 命令别名的实现

```typescript
register(command: Command): void {
    this.commands.set(command.name, command);
    
    // 别名也指向同一个命令对象
    if (command.aliases) {
        for (const alias of command.aliases) {
            this.commands.set(alias, command);
        }
    }
}

// 结果：
// commands.get('help') === commands.get('h')  // true
```

---

### 4. 命令解析

```typescript
parse(input: string): { name: string; args: string } | null {
    if (!input.startsWith('/')) return null;
    
    const trimmed = input.slice(1);  // 去掉 /
    const [name, ...argsParts] = trimmed.split(' ');
    const args = argsParts.join(' ');
    
    return { name, args };
}

// 例子：
// "/help abc def" → { name: "help", args: "abc def" }
// "/clear" → { name: "clear", args: "" }
```

### ✨ 双重历史的设计权衡

**为什么需要两套历史？**

1. **UI 消息**（`appStore.messages`）：
   - 简化的文本消息
   - 只有 role + text
   - 用于界面显示

2. **Agent 消息**（`depsAgent.getHistory()`）：
   - 完整的 LLM 消息
   - 包含 tool_calls、tool_result 等
   - 用于发给 LLM

**优点**：
- UI 简洁，不显示复杂的 tool 结构
- Agent 完整，LLM 能理解上下文

**缺点**：
- 需要同步两套数据
- 容易出现不一致

**Claude Code 的做法**：
- 也是双重历史
- 用事件系统同步
- 更复杂但更健壮
