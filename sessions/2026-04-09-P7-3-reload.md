## 📚 核心概念：两种系统提示词管理策略

### 策略 1：System 消息存在历史中

```typescript
// 初始化时添加到历史
deps.addMessage(createSystemMessage(systemPrompt));

// /reload 时替换第一条 system 消息
const history = deps.getHistory();
history[0] = createSystemMessage(newSystemPrompt);
deps.replaceHistory(history);
```

**特点**：
- ✅ 简单直观
- ❌ 系统消息占用历史空间
- ❌ 压缩时需要特殊处理
- ❌ reload 需要修改历史

---

### 策略 2：动态构建（Claude Code 的做法）✅

```typescript
// api.ts - 每次 API 调用时动态构建
async function sendMessage(messages: Message[]): Promise<...> {
    // 1. 读取最新的 CLAUDE.md
    const projectInstructions = loadProjectInstructions();
    
    // 2. 构建系统提示词
    const systemPrompt = buildSystemPrompt({
        cwd: process.cwd(),
        datetime: new Date().toISOString(),
        tools: toolRegistry.list(),
        projectInstructions  // 👈 包含最新内容
    });
    
    // 3. 临时添加 system 消息（不存到历史）
    const messagesWithSystem = [
        createSystemMessage(systemPrompt),
        ...messages
    ];
    
    // 4. 发送给 API
    await api.chat(messagesWithSystem);
}

// /reload 命令
// 什么都不用做！下次 API 调用会自动读取最新文件
```

**特点**：
- ✅ 历史消息更简洁（不含 system）
- ✅ reload 无需修改历史
- ✅ 系统提示词始终是最新的
- ✅ 压缩时不用担心 system 消息
- ⚠️ 每次调用都要读文件（性能影响可忽略）

## 🎯 实现的功能

### 1. `loadProjectInstructions()` - 读取 CLAUDE.md

```typescript
// src/utils/loadProjectInstructions.ts
export function loadProjectInstructions(cwd: string = process.cwd()): string | null {
    const filePath = join(cwd, 'CLAUDE.md');
    
    if (!existsSync(filePath)) {
        return null;
    }
    
    try {
        const content = readFileSync(filePath, 'utf-8');
        return content.trim();
    } catch (error) {
        console.error(`读取 CLAUDE.md 失败: ${error}`);
        return null;
    }
}
```

---

### 2. `buildSystemPrompt()` - 构建系统提示词

```typescript
// src/utils/buildSystemPrompt.ts
type SystemPromptContext = {
    cwd: string
    datetime: string
    tools: { name: string; description: string }[]
    memory?: string
    projectInstructions?: string  // 👈 新增
}

function buildSystemPrompt(context: SystemPromptContext): string {
    const { cwd, datetime, tools, memory, projectInstructions } = context;
    let prompt = `你是一个 AI 编程助手，帮助用户完成编程任务。\n\n`;
    
    // 项目指令（最高优先级，放在最前面）
    if (projectInstructions) {
        prompt += `## 项目指令\n\n${projectInstructions}\n\n`;
    }
    
    prompt += `## 环境信息\n\n`;
    prompt += `当前工作目录：${cwd}\n`;
    prompt += `当前时间：${datetime}\n\n`;
    
    prompt += `## 可用工具\n\n`;
    tools.forEach(tool => {
        prompt += `- ${tool.name}: ${tool.description}\n`;
    });
    
    if (memory) {
        prompt += `\n## 项目记忆\n\n${memory}\n`;
    }
    
    return prompt;
}
```

### 5. `/reload` 命令

```typescript
// src/commands/reload.ts
export const reloadCommand: Command = {
    name: 'reload',
    description: '重新加载 CLAUDE.md 项目指令',
    async execute(args: string): Promise<string> {
        const filePath = join(process.cwd(), 'CLAUDE.md');
        
        if (!existsSync(filePath)) {
            return '❌ 未找到 CLAUDE.md 文件';
        }
        
        const content = loadProjectInstructions();
        
        if (!content) {
            return '❌ CLAUDE.md 读取失败';
        }
        
        const lines = content.split('\n').length;
        const chars = content.length;
        
        return `✅ CLAUDE.md 已重新加载\n\n` +
               `文件路径：${filePath}\n` +
               `内容大小：${chars} 字符，${lines} 行\n\n` +
               `💡 提示：新的指令将在下次对话时生效`;
    }
};
```

## 🎓 关键学习点

### 1. Claude Code 的系统提示词管理策略
- **不把 system 消息存在历史中**
- **每次 API 调用时动态构建**
- **reload 无需修改历史**

### 2. 为什么这样设计？
- ✅ 历史消息更简洁
- ✅ 系统提示词始终最新
- ✅ 压缩时不用特殊处理
- ✅ 符合"单一职责原则"：历史只管对话，系统提示词由 API 层管理

### 3. 架构优势
```
历史层 (history.ts)     → 只存储对话消息（user/assistant/tool）
  ↓
API 层 (api.ts)         → 动态构建系统提示词 + 发送请求
  ↓
LLM API                 → 接收完整消息（system + history）
```
